- 대답은 항상 한국말로 할 것
- 모르겠거나 정보가 부족하면 어디가 부족한지 얘기해서 요구할것.
- 이슈번호는 항상 작성하되 별다른 언급이 없으면 최근 커밋의 이슈번호 사용
- @.claude/script/export_diff.sh 하고   @.claude/script/export_recent_commits.sh 을 사용해서 그 결과값으로 나온 데이터를 기반으로 커밋메시지 작성
- 커밋 메시지의 description은 너무 자세하게 말고 가능하면 한줄로 작성해.
- 마지막 대답은 아래고 같은 형식으로 할 것
```text
feat: 담당자 연락처 입력 개선 및 gitignore 업데이트 (#56)

- 담당자 연락처 필드에 숫자만 입력 가능하도록 개선
- 최대 11자리 입력 제한 및 실시간 포맷팅 적용
- maxLength, autoComplete, inputMode 속성 추가로 사용성 향상
```
- 확인되지 않는 파일이 있는 경우에는 프로젝트에 직접 접근해서 확인은 해도 되지만 그 어떤 수정도 일어나서는 안 됨
- 직접 커밋하지마. 그냥 나한테 알려줘. 커밋은 내가 직접 할거야
